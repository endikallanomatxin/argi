# 0.1

Goals:
- Basic language features are well defined and implemented.
- Core library has basic utilities.
- Language tooling supports `build`, `test`, `lsp`, `audit` and `format`
commands.

with tests covering the implemented features and protecting against
regressions.

## Language features

- Types
  - basic scalar types
  - aliases
  - explicit casts and basic type checks
  - 🔧 Structs
    - struct declaration
    - field access
    - `init` / `deinit`
    - public/private boundary where already supported
  - 🔧 Choice / nullability / errors
    - `choice` values with payloads
    - basic optional/nullability flow
    - error unions and unwrapping
    - basic propagation ergonomics

- ✅ Variables and constants
  - local bindings
  - mutable and immutable bindings
  - clear initialization rules

- 🔧 Functions
  - function declaration
  - named arguments and defaults
  - multiple dispatch baseline
  - return values

- 🔧 Modules
  - folder-based modules
  - `#import(...)` assigned to a name
  - current/parent/project-root resolution
  - bare imports through `more/`

- ✅ Control flow
  - `if` / `else`
  - `while`
  - `for` with iterators
  - `match`

- 📝 Basic polymorphism
  - compiletime parameters / generics baseline
  - multiple dispatch kept coherent with generics
  - advanced abstract/virtual features stay out of 0.1

- 💭 Memory management.


## Core library

- System
  - ✅ Implement a minimal `System` system struct to encapsulate system
  interactions and resources.
  - 💭 Ensure `System` elements can only be instatiated once.
    Maybe create a new keyword `once` to enforce this at compile time.

- 📝 Strings
  - basic owned string
  - basic borrowed string view
  - C string interop

- 🔧 IO
  - `File`, `Reader`, `Writer`
  - `stdin`, `stdout`, `stderr`
  - byte IO
  - minimal text helpers needed for compiler-style programs

- 📝 Memory
  - minimal allocator interface
  - arena / bump allocator
  - ownership of allocator-backed buffers kept explicit

- 📝 Collections
  - array baseline
  - slice / view baseline
  - one dynamic list/array type good enough for compiler work

- 💭 Testing
  - minimal assertion helpers
  - minimal test helpers for core and compiler code
  - diagnostics helpers good enough for compiler errors
